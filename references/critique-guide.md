# Design Critique In-Depth Guide

> Detailed reference for Phase 7. Provides scoring criteria, scene-specific focus areas, and a common issues checklist.

---

## Scoring Criteria Explained

### 1. Philosophy Alignment

| Score | Criteria |
|-------|---------|
| 9-10 | The design perfectly embodies the core spirit of the chosen philosophy; every detail has philosophical grounding |
| 7-8 | Overall direction is correct, key characteristics are in place, with occasional detail deviations |
| 5-6 | Intent is visible, but execution mixes in other stylistic elements — not pure enough |
| 3-4 | Surface-level imitation only; the philosophical core is not understood |
| 1-2 | Essentially unrelated to the chosen philosophy |

**Review checkpoints**:
- Does it use the signature techniques of that designer / studio?
- Do the color, typography, and layout align with that philosophical system?
- Are there any "self-contradictory" elements? (e.g., choosing Kenya Hara but cramming the layout full)

### 2. Visual Hierarchy

| Score | Criteria |
|-------|---------|
| 9-10 | The viewer's eye flows naturally along the designer's intended path; information is absorbed with zero friction |
| 7-8 | Primary/secondary relationships are clear, with 1-2 occasional hierarchy ambiguities |
| 5-6 | Can distinguish heading from body, but middle-level hierarchy is confused |
| 3-4 | Information is laid flat with no clear visual entry point |
| 1-2 | Chaotic — the viewer doesn't know where to look first |

**Review checkpoints**:
- Is the contrast between heading and body font size sufficient? (At least 2.5×)
- Do color / weight / size establish 3-4 distinct levels?
- Is whitespace actively guiding the eye?
- "Squint test": squint your eyes — is the hierarchy still clear?

### 3. Craft Quality

| Score | Criteria |
|-------|---------|
| 9-10 | Pixel-perfect: alignment, spacing, and color have zero flaws |
| 7-8 | Overall polished, with 1-2 minor alignment/spacing issues |
| 5-6 | Generally aligned, but spacing is inconsistent and color use is unsystematic |
| 3-4 | Obvious alignment errors, chaotic spacing, too many colors |
| 1-2 | Rough — looks like a draft |

**Review checkpoints**:
- Is a consistent spacing system used (e.g., 8pt grid)?
- Is spacing between similar elements consistent?
- Is the number of colors controlled? (Usually no more than 3-4)
- Is the type family consistent? (Usually no more than 2)
- Is edge alignment precise?

### 4. Functionality

| Score | Criteria |
|-------|---------|
| 9-10 | Every design element serves the goal; zero redundancy |
| 7-8 | Functional intent is clear; a small amount of decoration could be removed |
| 5-6 | Basically functional, but obvious decorative elements distract attention |
| 3-4 | Form over function; the user has to work to find information |
| 1-2 | Completely overwhelmed by decoration; has lost the ability to communicate information |

**Review checkpoints**:
- If any element is removed, does the design get worse? (If not, it should be removed)
- Are the CTA / key messages in the most prominent positions?
- Are there any elements added "just because they look good"?
- Does information density match the medium? (PPT should not be too dense; PDFs can be denser)

### 5. Originality

| Score | Criteria |
|-------|---------|
| 9-10 | Refreshing — found a unique expression within the philosophical framework |
| 7-8 | Has its own ideas; not a simple template application |
| 5-6 | By the book — looks like a template |
| 3-4 | Heavy use of clichés (e.g., a gradient sphere to represent AI) |
| 1-2 | Completely a template or stock asset collage |

**Review checkpoints**:
- Are common clichés avoided? (See the "Common Issues" checklist below)
- Is there personal expression while adhering to the design philosophy?
- Are there any "unexpected but perfectly logical" design decisions?

## Scene-Specific Review Focus

Different output types have different review priorities:

| Scene | Most Important | Second Priority | Can Be Relaxed |
|-------|---------------|-----------------|----------------|
| Social media cover / article image | Originality, Visual Hierarchy | Philosophy Alignment | Functionality (single image, no interaction) |
| Infographic | Functionality, Visual Hierarchy | Craft Quality | Originality (accuracy first) |
| PPT / Keynote | Visual Hierarchy, Functionality | Craft Quality | Originality (clarity first) |
| PDF / White paper | Craft Quality, Functionality | Visual Hierarchy | Originality (professionalism first) |
| Landing page / website | Functionality, Visual Hierarchy | Originality | — (all-round requirement) |
| App UI | Functionality, Craft Quality | Visual Hierarchy | Philosophy Alignment (usability first) |
| Xiaohongshu image | Originality, Visual Hierarchy | Philosophy Alignment | Craft Quality (atmosphere first) |

## Top 10 Common Design Issues

### 1. AI Tech Clichés
**Problem**: Gradient spheres, digital rain, blue circuit boards, robot faces
**Why it's a problem**: Users are visually fatigued by these; they can't distinguish you from everyone else
**Fix**: Replace literal symbols with abstract metaphors (e.g., use a "conversation" metaphor instead of a chat bubble icon)

### 2. Insufficient Font Size Hierarchy
**Problem**: The gap between heading and body is too small (<2.5×)
**Why it's a problem**: Users cannot quickly locate key information
**Fix**: Headings should be at least 3× the body size (e.g., body 16px → heading 48-64px)

### 3. Too Many Colors
**Problem**: Using 5+ colors with no primary/secondary distinction
**Why it's a problem**: Visual confusion, weak brand identity
**Fix**: Limit to 1 primary + 1 secondary + 1 accent + grayscale

### 4. Inconsistent Spacing
**Problem**: Element spacing is arbitrary with no system
**Why it's a problem**: Looks unprofessional; visual rhythm is chaotic
**Fix**: Establish an 8pt grid system (spacing values: 8/16/24/32/48/64px only)

### 5. Insufficient Whitespace
**Problem**: All space is filled with content
**Why it's a problem**: Information overcrowding causes reading fatigue and actually reduces information delivery efficiency
**Fix**: Whitespace should occupy at least 40% of the total area (60%+ for minimalist styles)

### 6. Too Many Typefaces
**Problem**: Using 3+ fonts
**Why it's a problem**: Visual noise; undermines cohesion
**Fix**: Maximum 2 typefaces (1 for headings + 1 for body); create variation with weight and size

### 7. Inconsistent Alignment
**Problem**: Some elements are left-aligned, some centered, some right-aligned
**Why it's a problem**: Destroys the sense of visual order
**Fix**: Choose one alignment (left-align is recommended) and apply it globally

### 8. Decoration Overwhelming Content
**Problem**: Background patterns / gradients / shadows steal focus from the main content
**Why it's a problem**: Priorities inverted — users came to read information, not admire decoration
**Fix**: "If I remove this decoration, does the design get worse?" If not, remove it

### 9. Cyber Neon Overuse
**Problem**: Dark blue background (#0D1117) + neon glowing effects
**Why it's a problem**: This is a default aesthetic no-go zone for this skill (a baseline of taste), and has also become one of the biggest clichés — users can override for their own brand
**Fix**: Choose a more distinctive color scheme (reference the 20 styles' color systems)

### 10. Information Density Mismatched to Medium
**Problem**: A full page of text in a PPT slide / 10 elements crammed into a cover image
**Why it's a problem**: Different media have different optimal information densities
**Fix**:
- PPT: 1 core idea per slide
- Cover image: 1 visual focal point
- Infographic: layered presentation
- PDF: can be denser, but requires clear navigation

- Visual Hierarchy: X/10 [one-sentence summary]
- Craft Quality: X/10 [one-sentence summary]
- Functionality: X/10 [one-sentence summary]
- Originality: X/10 [one-sentence summary]

### Strengths (Keep)
- [Specifically identify what works well, described in design language]

### Issues (Fix)
[Ordered by severity]

**1. [Issue Name]** — ⚠️ Critical / ⚡ Important / 💡 Optimization
- Current: [describe the current state]
- Problem: [why this is a problem]
- Fix: [specific action to take, including numerical values]
