# Design Critique In-Depth Guide

> Detailed reference for Phase 7. Scoring criteria, scene-specific focus areas, and common issues checklist.

---

## Scoring Criteria Explained

### 1. Philosophy Alignment

| Score | Criteria |
|-------|---------|
| 9-10 | Design perfectly embodies chosen philosophy; every detail has philosophical grounding |
| 7-8 | Overall direction correct, key characteristics in place, occasional detail deviations |
| 5-6 | Intent visible, but execution mixes other stylistic elements — not pure enough |
| 3-4 | Surface-level imitation only; philosophical core not understood |
| 1-2 | Essentially unrelated to chosen philosophy |

**Review checkpoints**:
- Uses signature techniques of that designer / studio?
- Color, typography, layout align with that philosophical system?
- Any "self-contradictory" elements? (e.g., chose Kenya Hara but crammed layout full)

### 2. Visual Hierarchy

| Score | Criteria |
|-------|---------|
| 9-10 | Eye flows naturally along designer's intended path; information absorbed with zero friction |
| 7-8 | Primary/secondary relationships clear; 1-2 occasional hierarchy ambiguities |
| 5-6 | Can distinguish heading from body, but mid-level hierarchy is confused |
| 3-4 | Information laid flat with no clear visual entry point |
| 1-2 | Chaotic — viewer doesn't know where to look first |

**Review checkpoints**:
- Heading vs. body font size contrast sufficient? (At least 2.5×)
- Color / weight / size establish 3-4 distinct levels?
- Whitespace actively guiding eye?
- "Squint test": squint — is hierarchy still clear?

### 3. Craft Quality

| Score | Criteria |
|-------|---------|
| 9-10 | Pixel-perfect: alignment, spacing, color have zero flaws |
| 7-8 | Overall polished; 1-2 minor alignment/spacing issues |
| 5-6 | Generally aligned; spacing inconsistent, color use unsystematic |
| 3-4 | Obvious alignment errors, chaotic spacing, too many colors |
| 1-2 | Rough — looks like draft |

**Review checkpoints**:
- Consistent spacing system used (e.g., 8pt grid)?
- Spacing between similar elements consistent?
- Colors controlled? (Usually no more than 3-4)
- Type family consistent? (Usually no more than 2)
- Edge alignment precise?

### 4. Functionality

| Score | Criteria |
|-------|---------|
| 9-10 | Every element serves goal; zero redundancy |
| 7-8 | Functional intent clear; small amount of decoration could be removed |
| 5-6 | Basically functional; obvious decorative elements distract |
| 3-4 | Form over function; user has to work to find information |
| 1-2 | Overwhelmed by decoration; lost ability to communicate information |

**Review checkpoints**:
- Remove any element — does design get worse? (If not, remove it)
- CTA / key messages in most prominent positions?
- Elements added "just because they look good"?
- Information density matches medium? (PPT not too dense; PDFs can be denser)

### 5. Originality

| Score | Criteria |
|-------|---------|
| 9-10 | Refreshing — found unique expression within philosophical framework |
| 7-8 | Has own ideas; not a simple template application |
| 5-6 | By the book — looks like template |
| 3-4 | Heavy use of clichés (e.g., gradient sphere to represent AI) |
| 1-2 | Entirely template or stock asset collage |

**Review checkpoints**:
- Common clichés avoided? (See "Common Issues" checklist below)
- Personal expression while adhering to design philosophy?
- Any "unexpected but perfectly logical" design decisions?

## Scene-Specific Review Focus

| Scene | Most Important | Second Priority | Can Be Relaxed |
|-------|---------------|-----------------|----------------|
| Social media cover / article image | Originality, Visual Hierarchy | Philosophy Alignment | Functionality (single image, no interaction) |
| Infographic | Functionality, Visual Hierarchy | Craft Quality | Originality (accuracy first) |
| PPT / Keynote | Visual Hierarchy, Functionality | Craft Quality | Originality (clarity first) |
| PDF / White paper | Craft Quality, Functionality | Visual Hierarchy | Originality (professionalism first) |
| Landing page / website | Functionality, Visual Hierarchy | Originality | — (all-round requirement) |
| App UI | Functionality, Craft Quality | Visual Hierarchy | Philosophy Alignment (usability first) |
| Xiaohongshu image | Originality, Visual Hierarchy | Philosophy Alignment | Craft Quality (atmosphere first) |

## Top 10 Common Design Issues

### 1. AI Tech Clichés
**Problem**: Gradient spheres, digital rain, blue circuit boards, robot faces
**Why**: Users visually fatigued; can't distinguish you from everyone else
**Fix**: Replace literal symbols with abstract metaphors (e.g., "conversation" metaphor instead of chat bubble)

### 2. Insufficient Font Size Hierarchy
**Problem**: Gap between heading and body too small (<2.5×)
**Why**: Users can't quickly locate key information
**Fix**: Headings at least 3× body size (e.g., body 16px → heading 48-64px)

### 3. Too Many Colors
**Problem**: 5+ colors with no primary/secondary distinction
**Why**: Visual confusion, weak brand identity
**Fix**: Limit to 1 primary + 1 secondary + 1 accent + grayscale

### 4. Inconsistent Spacing
**Problem**: Element spacing arbitrary with no system
**Why**: Looks unprofessional; visual rhythm chaotic
**Fix**: 8pt grid system (values: 8/16/24/32/48/64px only)

### 5. Insufficient Whitespace
**Problem**: All space filled with content
**Why**: Information overcrowding causes reading fatigue; actually reduces delivery efficiency
**Fix**: Whitespace at least 40% of total area (60%+ for minimalist styles)

### 6. Too Many Typefaces
**Problem**: 3+ fonts
**Why**: Visual noise; undermines cohesion
**Fix**: Max 2 typefaces (1 heading + 1 body); create variation with weight and size

### 7. Inconsistent Alignment
**Problem**: Some elements left-, some center-, some right-aligned
**Why**: Destroys visual order
**Fix**: Choose one alignment (left-align recommended) and apply globally

### 8. Decoration Overwhelming Content
**Problem**: Background patterns / gradients / shadows steal focus from main content
**Why**: Priorities inverted — users came to read information, not admire decoration
**Fix**: "If I remove this decoration, does design get worse?" If not, remove it

### 9. Cyber Neon Overuse
**Problem**: Dark blue background (#0D1117) + neon glowing effects
**Why**: Default aesthetic no-go zone for this skill; also one of the biggest clichés — users can override for own brand
**Fix**: Choose more distinctive color scheme (reference 20 styles' color systems)

### 10. Information Density Mismatched to Medium
**Problem**: Full page of text in PPT slide / 10 elements crammed into cover image
**Why**: Different media have different optimal information densities
**Fix**:
- PPT: 1 core idea per slide
- Cover image: 1 visual focal point
- Infographic: layered presentation
- PDF: can be denser, but requires clear navigation

```
## Design Critique Report

**Overall Score**: X.X/10 [Excellent (8+) / Good (6-7.9) / Needs Improvement (4-5.9) / Failing (<4)]

**Breakdown**:
- Philosophy Alignment: X/10 [one-sentence summary]
- Visual Hierarchy: X/10 [one-sentence summary]
- Craft Quality: X/10 [one-sentence summary]
- Functionality: X/10 [one-sentence summary]
- Originality: X/10 [one-sentence summary]

### Strengths (Keep)
- [Specifically identify what works well, described in design language]

### Issues (Fix)
[Ordered by severity]

**1. [Issue Name]** — ⚠️ Critical / ⚡ Important / 💡 Optimization
- Current: [describe current state]
- Problem: [why this is a problem]
- Fix: [specific action, including numerical values]

### Quick Wins
If you only have 5 minutes, prioritize these 3:
- [ ] [Most impactful fix]
- [ ] [Second most important fix]
- [ ] [Third most important fix]
```
